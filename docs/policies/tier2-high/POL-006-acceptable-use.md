## 1. Purpose

This Acceptable Use Policy (AUP) establishes guidelines for the proper use of GreenLang Climate OS information technology resources, including computing equipment, software, networks, email, messaging systems, cloud services, and development environments. The policy aims to protect GreenLang's assets, reputation, and data while enabling employees and contractors to perform their duties effectively.

This policy ensures that:
- Company resources are used productively and securely
- Intellectual property and confidential information are protected
- Legal and regulatory compliance requirements are met
- A respectful and professional work environment is maintained
- Security risks from misuse are minimized

---

## 2. Scope

### 2.1 Applicability

This policy applies to:
- All GreenLang employees (full-time, part-time, temporary)
- Contractors, consultants, and third-party personnel
- Interns and volunteers with access to GreenLang systems
- Any individual using GreenLang-owned or managed IT resources

### 2.2 Covered Resources

This policy covers the use of:
- Company-provided computers, laptops, and mobile devices
- Email systems and communication platforms (Slack, Teams, etc.)
- Cloud services (AWS, Azure, GCP, SaaS applications)
- Development environments (IDEs, repositories, CI/CD pipelines)
- Internal networks and VPN connections
- Internet access through company resources
- Software licensed by GreenLang
- Data storage systems (local, network, cloud)
- Bring Your Own Device (BYOD) equipment accessing company resources

---

## 3. Policy Statement

### 3.1 General Principles

All users of GreenLang IT resources shall:
- Use resources primarily for legitimate business purposes
- Protect confidential information and company assets
- Comply with all applicable laws, regulations, and company policies
- Exercise reasonable judgment regarding personal use
- Report security incidents or policy violations promptly
- Cooperate with IT and Security teams during investigations

### 3.2 Acceptable Use of Company Systems

#### 3.2.1 Email and Messaging Systems

Users may use email and messaging systems for:
- Business communications with colleagues, customers, and partners
- Scheduling meetings and coordinating work activities
- Sharing work-related documents and information
- Participating in approved distribution lists and channels

Email and messaging requirements:
- Use professional language and tone in all communications
- Include appropriate disclaimers for external communications
- Verify recipient addresses before sending sensitive information
- Archive important communications per retention requirements
- Never auto-forward company email to personal accounts

#### 3.2.2 Cloud Services

Approved cloud services include:
- AWS (primary cloud infrastructure)
- Azure AD (identity management)
- GitHub (source code management)
- Confluence (documentation)
- Jira (project management)
- Slack (team communication)

Users shall:
- Access only cloud services approved by IT Security
- Use single sign-on (SSO) where available
- Enable multi-factor authentication (MFA) on all accounts
- Not store Restricted or Confidential data on unapproved services
- Report shadow IT discoveries to the Security team

#### 3.2.3 Development Environments

Developers shall:
- Use approved IDEs, tools, and libraries
- Follow secure coding standards (OWASP Top 10)
- Commit code only to authorized repositories
- Never store secrets, credentials, or API keys in code
- Use pre-commit hooks for security scanning
- Participate in code reviews before merging

### 3.3 Personal Use Guidelines

GreenLang permits limited personal use of IT resources under the following conditions:

#### 3.3.1 De Minimis Personal Use

Acceptable personal use:
- Brief personal communications during breaks
- Occasional internet browsing during non-work time
- Limited personal file storage (not exceeding 1GB)
- Access to personal email during breaks

Personal use must:
- Not interfere with job performance
- Not consume excessive bandwidth or storage
- Not expose the company to legal liability
- Not violate any provisions of this policy
- Not involve prohibited activities

#### 3.3.2 Restrictions on Personal Use

Personal use shall not:
- Occur during critical work periods or deadlines
- Impact network performance for other users
- Include downloading large media files
- Involve use of company resources for personal business
- Include storing personal data classified as sensitive

### 3.4 Prohibited Activities

The following activities are strictly prohibited on GreenLang IT resources:

#### 3.4.1 Illegal Activities

- Unauthorized access to systems or data (hacking)
- Distribution of pirated software, music, or media
- Copyright or trademark infringement
- Fraud, identity theft, or financial crimes
- Distribution of malware or malicious code
- Violations of export control regulations
- Child exploitation material (zero tolerance)

#### 3.4.2 Harassment and Discrimination

- Sending harassing, threatening, or abusive messages
- Creating or distributing discriminatory content
- Bullying, intimidation, or stalking
- Creating a hostile work environment
- Sexual harassment in any form
- Hate speech or offensive content based on protected characteristics

#### 3.4.3 Unauthorized Software and Services

- Installing unapproved software on company devices
- Using unauthorized cloud storage services (Dropbox, personal Google Drive)
- Running file-sharing applications (BitTorrent, P2P)
- Installing browser extensions without IT approval
- Using unauthorized remote access tools

#### 3.4.4 Cryptomining

- Using company resources for cryptocurrency mining
- Installing mining software on any company system
- Using company electricity or compute for personal mining
- Participating in mining pools using company resources

#### 3.4.5 Circumventing Security Controls

- Disabling antivirus or endpoint protection
- Bypassing network security controls or firewalls
- Using proxies or VPNs to evade monitoring
- Sharing credentials or access tokens
- Tailgating into secure areas
- Tampering with audit logs or monitoring systems

#### 3.4.6 Other Prohibited Activities

- Gambling or betting using company resources
- Accessing pornographic or explicit material
- Operating personal businesses using company resources
- Sending spam or chain letters
- Impersonating another user or entity
- Unauthorized data exfiltration
- Intentionally degrading system performance

### 3.5 Email and Communication Standards

#### 3.5.1 Professional Standards

All communications shall:
- Be professional, respectful, and appropriate
- Use clear and concise language
- Include relevant context and purpose
- Respect recipients' time and attention
- Avoid ALL CAPS (interpreted as shouting)

#### 3.5.2 Confidentiality

When communicating:
- Classify information appropriately (Public, Internal, Confidential, Restricted)
- Use encryption for Confidential and Restricted data
- Verify recipients before sharing sensitive information
- Never discuss confidential matters in public channels
- Use BCC appropriately for large distribution lists

#### 3.5.3 External Communications

When communicating externally:
- Use company email for all business communications
- Include the standard email disclaimer
- Obtain approval before making commitments
- Do not disparage competitors
- Report unsolicited contact from potential bad actors

### 3.6 Social Media Guidelines

#### 3.6.1 Personal Social Media

Employees using personal social media:
- Shall not disclose Confidential or Restricted company information
- Shall not claim to speak on behalf of GreenLang without authorization
- Shall not share proprietary methodologies or algorithms
- Shall use appropriate privacy settings
- Shall include disclaimers when discussing work-related topics
- Shall not post disparaging content about the company, colleagues, or customers

#### 3.6.2 Official Company Social Media

Only authorized personnel may:
- Post on official GreenLang social media accounts
- Respond to customer inquiries via social platforms
- Make public statements about company products or services
- Engage with media or analysts on company matters

#### 3.6.3 Prohibited Social Media Activities

- Sharing screenshots of internal systems or data
- Discussing unreleased products or features
- Sharing customer names or data without consent
- Responding to security incidents on social media
- Engaging in arguments or heated debates representing the company

### 3.7 Intellectual Property Protection

#### 3.7.1 Company Intellectual Property

Users shall:
- Protect GreenLang's intellectual property (code, algorithms, designs, data)
- Use company IP only for authorized business purposes
- Not copy, share, or distribute proprietary materials externally
- Return all IP upon termination of employment
- Report suspected IP theft or infringement

#### 3.7.2 Third-Party Intellectual Property

Users shall:
- Respect copyrights, trademarks, and patents
- Only use properly licensed software and content
- Attribute sources appropriately
- Not reverse engineer third-party software
- Report license compliance concerns

### 3.8 Software Licensing Compliance

#### 3.8.1 License Management

GreenLang maintains compliance with all software licenses:
- Only install software with valid licenses
- Do not exceed license seat counts
- Report software needs to IT for proper procurement
- Do not use trial software for production purposes
- Maintain records of all installed software

#### 3.8.2 Open Source Compliance

When using open source software:
- Verify license compatibility (Apache, MIT, GPL, etc.)
- Maintain attribution as required by licenses
- Do not incorporate GPL-licensed code into proprietary products
- Report open source usage to the Legal team
- Follow the dependency management policy

### 3.9 Bring Your Own Device (BYOD) Policy

#### 3.9.1 Approved Devices

Personal devices may access company resources only if:
- The device is on the approved device list
- Mobile Device Management (MDM) is installed and active
- The device meets minimum security requirements
- The user has signed the BYOD agreement

#### 3.9.2 MDM Requirements

All BYOD devices must have:
- Remote wipe capability enabled
- Device encryption enabled
- Screen lock with minimum 6-digit PIN or biometric
- Latest operating system security patches
- No jailbreaking or rooting

#### 3.9.3 BYOD Restrictions

Personal devices shall not:
- Store Restricted data locally
- Access production systems directly
- Use personal cloud services for company data
- Connect to the internal network without VPN

#### 3.9.4 Company Rights

GreenLang reserves the right to:
- Remotely wipe company data from personal devices
- Audit BYOD devices for compliance
- Revoke BYOD privileges for policy violations
- Require immediate device return upon termination

### 3.10 Monitoring and Privacy Notice

#### 3.10.1 Monitoring Statement

GreenLang monitors IT resources to ensure:
- Policy compliance and security
- Protection of company assets and data
- Network performance and availability
- Legal and regulatory compliance
- Incident detection and response

#### 3.10.2 What Is Monitored

Monitoring may include:
- Network traffic and bandwidth usage
- Email metadata (not content unless warranted)
- Web browsing activity and URLs visited
- File access and transfer logs
- Authentication and access logs
- Endpoint security alerts

#### 3.10.3 Privacy Expectations

Users should have no expectation of privacy when:
- Using company-owned devices
- Accessing company networks
- Using company email or messaging systems
- Storing data on company systems

Personal use activities on company resources are also subject to monitoring.

#### 3.10.4 Investigation Authority

GreenLang may conduct investigations with proper authorization:
- Access email content with Legal and HR approval
- Examine files on company devices
- Review communication logs
- Analyze system access patterns
- Engage forensic specialists when necessary

## 4. Roles and Responsibilities

### 4.1 All Users

- Read, understand, and comply with this policy
- Complete annual security awareness training
- Report suspected violations or security incidents
- Protect credentials and access tokens
- Lock workstations when away from desk

### 4.2 Managers

- Ensure team members understand this policy
- Address policy questions and concerns
- Report suspected violations to HR and Security
- Model appropriate behavior
- Support policy enforcement

### 4.3 IT Department

- Implement technical controls supporting this policy
- Maintain approved software and service lists
- Monitor systems for policy violations
- Respond to security incidents
- Provide guidance on acceptable use questions

### 4.4 Information Security

- Define security requirements for IT resources
- Conduct monitoring and auditing
- Investigate potential policy violations
- Update policy based on emerging threats
- Provide security awareness training

### 4.5 Human Resources

- Include policy in onboarding process
- Maintain acknowledgment records
- Address policy violations through disciplinary process
- Coordinate termination procedures
- Support investigations as needed

### 4.6 Legal Department

- Review policy for legal compliance
- Advise on intellectual property matters
- Support investigations requiring legal oversight
- Coordinate with law enforcement if necessary
- Review external communications for compliance

## 5. Procedures

### 5.1 Policy Acknowledgment

1. All new employees acknowledge this policy during onboarding
2. Annual re-acknowledgment is required for all users
3. Acknowledgments are recorded in the HR system
4. Failure to acknowledge may result in access restriction

### 5.2 Reporting Violations

1. Report suspected violations to Security (security@greenlang.io) or HR
2. Anonymous reporting is available via the ethics hotline
3. Do not attempt to investigate violations independently
4. Preserve evidence by not modifying systems or logs
5. Cooperate fully with authorized investigations

### 5.3 Requesting Exceptions

1. Submit exception requests to the CISO via the exception request form
2. Provide business justification and risk assessment
3. Identify compensating controls if applicable
4. Await approval before proceeding
5. Exceptions are time-limited and subject to review

### 5.4 Software Installation Request

1. Submit software request through IT Service Portal
2. IT Security reviews for security and licensing
3. Approved software is added to the catalog
4. Installation performed by IT or through approved channels
5. Unapproved software will be removed without notice

---

## 6. Exceptions

Exceptions to this policy may be granted under specific circumstances:

### 6.1 Exception Criteria

Exceptions may be considered when:
- Business need cannot be met through standard means
- Compensating controls adequately address risks
- Exception is time-limited and scoped
- Risk is formally accepted by appropriate authority

### 6.2 Exception Process

1. Request submitted via GRC tool or email to CISO
2. Security team assesses risk and compensating controls
3. Approval authority based on risk level:
   - Low risk: Security Manager
   - Medium risk: CISO
   - High risk: CISO + CTO
4. Exceptions documented and tracked
5. Annual review of all active exceptions

### 6.3 Non-Negotiable Requirements

The following have no exceptions:
- Illegal activities
- Harassment and discrimination
- Child exploitation material
- Intentional security control circumvention
- Credential sharing

---

## 7. Enforcement

### 7.1 Violation Consequences

Violations of this policy may result in:
- Verbal warning for minor first-time offenses
- Written warning with documentation
- Suspension of system access
- Termination of employment
- Civil legal action for damages
- Criminal prosecution for illegal activities

### 7.2 Progressive Discipline

GreenLang follows progressive discipline:
1. **First Violation (Minor):** Verbal warning and retraining
2. **Second Violation or Moderate Severity:** Written warning
3. **Third Violation or Serious Severity:** Final written warning, possible suspension
4. **Fourth Violation or Severe Violation:** Termination

Severe violations (illegal activities, intentional data breach, harassment) may result in immediate termination.

### 7.3 Appeal Process

Employees may appeal disciplinary actions through:
1. Discussion with immediate manager
2. Escalation to HR Business Partner
3. Formal appeal to HR Director
4. Final appeal to CHRO (decision is final)

## 9. Definitions

| Term | Definition |
|------|------------|
| **BYOD** | Bring Your Own Device - personal devices used for work purposes |
| **De Minimis** | Minimal personal use that does not materially impact work or resources |
| **MDM** | Mobile Device Management - software to manage and secure mobile devices |
| **Shadow IT** | IT systems, devices, or applications used without explicit IT approval |
| **SSO** | Single Sign-On - authentication allowing one login for multiple systems |
| **MFA** | Multi-Factor Authentication - requiring multiple verification methods |
| **Confidential** | Information that could harm the company if disclosed |
| **Restricted** | Highest sensitivity - PII, credentials, encryption keys |
| **Cryptomining** | Using computing resources to generate cryptocurrency |
| **Intellectual Property** | Creations of the mind including inventions, designs, and code |
