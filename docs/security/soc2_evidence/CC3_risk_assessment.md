# CC3 — Risk Assessment

**Owner:** CISO. **Review cadence:** annual (plus ad-hoc for material changes).

## Controls

### CC3.1 — Risk identification and analysis

- Evidence: annual risk register; Factors API threat model (STRIDE).
- Collection: manual.
- Artifacts:
  - [ ] `docs/security/STRIDE-THREAT-MODEL-FACTORS.md` (current revision).
  - [ ] Risk register spreadsheet (Security drive).
  - [ ] Risk appetite statement (signed CEO + CISO).

### CC3.2 — Fraud risk

- Evidence: fraud-risk workshop notes; billing-flow anti-fraud review.
- Collection: manual.

### CC3.3 — Change identification

- Evidence: architecture decision records (ADRs) for material changes.
- Collection: automated (`docs/adr/`).

### CC3.4 — Risk response

- Evidence: mitigation plan per Critical / High risk.
- Collection: manual.
