## 1.3 What Makes GreenLang Unique

### Differentiator 1: Zero-Hallucination Architecture

**The Problem with Competitors:**

Most carbon accounting platforms use LLMs (GPT-4, Claude) for calculations:
- **Non-deterministic**: Same input can produce different outputs
- **Hallucination risk**: LLMs confidently provide wrong emission factors
- **Audit failures**: 30%+ rejection rate when regulators verify
- **Legal liability**: Incorrect emissions reporting = securities fraud risk

**GreenLang's Approach:**

```
+-------------------------------------------------------------+
|                 ZERO-HALLUCINATION DESIGN                    |
|                                                              |
|  LLMs ALLOWED for:          LLMs NEVER used for:            |
|  - Document classification   - Emission factor lookup        |
|  - Entity name matching      - Numeric calculations          |
|  - Report narrative          - Unit conversions              |
|  - Recommendations           - Aggregation/summation         |
|  - User explanations         - Regulatory compliance         |
|                                                              |
|  RESULT: AI for intelligence, deterministic for numbers     |
+-------------------------------------------------------------+
```

**Implementation:**
- **Calculation engine**: Pure Python arithmetic (no LLM in path)
- **Factor broker**: Database lookups only (cascading: ecoinvent -> DEFRA -> EPA -> proxy)
- **Provenance**: SHA-256 hash of every calculation step
- **Verification**: Same input -> Same output, 100% reproducible

**Audit Results:**
- GreenLang: **95%+ pass rate** with Big 4 auditors
- Competitors: ~70% pass rate (failed audits = re-work + delays)
