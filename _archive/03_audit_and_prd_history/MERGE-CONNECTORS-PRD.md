# Technical Debt: Merge Similar Data Connectors

## Context
The `greenlang/agents/data/` directory has 15 standalone `*_agent.py` files (Layer 1) and 20 directory-based agent modules (Layer 2/SDK). Several connectors have significant code duplication in enums, models, connection configs, and data point structures. This PRD defines tasks to deduplicate by extracting shared code into shared modules.

## Architecture Rules
- All models use `GreenLangBase` from `greenlang.schemas`
- Shared enums go in `greenlang.schemas.enums` or domain-specific `_shared.py` files
- Layer 1 agents provide core logic; Layer 2 directories provide SDK/engines
- When merging, the directory-based module is the canonical home; single-file agents get folded in
- ALWAYS preserve backward-compatible re-exports in `greenlang/agents/data/__init__.py`
- NEVER break existing imports from other modules
